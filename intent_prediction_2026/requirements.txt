# Create a virtual environment in this folder and install into it:
#     python3 -m venv .venv
#     source .venv/bin/activate
#     pip install -r requirements.txt
#
# What needs what:
#   data_collection/pose_data_collection.py   ->  ultralytics, opencv, numpy
#   feature_extraction/extract_features.py     ->  nothing (Python standard library only)
#
# -----------------------------------------------------------------------------
# JETSON ORIN NANO (the deployment target)
#   PyTorch must come from NVIDIA's Jetson wheels, NOT from PyPI. A plain
#   `pip install torch` pulls a wheel that will not run on the Jetson.
#
#   1. Flash JetPack 6.2+ (CUDA 12.6, cuDNN 9.3); it ships Python 3.10.
#   2. Install torch + torchvision FIRST, from the Jetson AI Lab index:
#        pip install torch torchvision --index-url https://pypi.jetson-ai-lab.dev/jp6/cu126
#      (or NVIDIA's official Jetson PyTorch wheels).
#   3. THEN install this file:
#        pip install -r requirements.txt
#      ultralytics needs torch, but since it is already installed from the
#      NVIDIA wheel, pip leaves it as-is.
#
#   OpenCV: JetPack ships an OpenCV build; if you rely on that you can drop the
#   opencv line below. The pip `opencv-python` wheel also works and is required
#   for the on-screen display window. Use `opencv-python-headless` only if you always
#   run headless (headless=True in main()).
#
#   TensorRT (.engine): tensorrt ships with JetPack. To export a .engine from a
#   .pt model you also need onnx + onnxruntime-gpu (see the optional block).
#
# MACOS / x86 LINUX (for testing the logic off-device)
#   Just `pip install -r requirements.txt`; ultralytics pulls a compatible torch
#   from PyPI automatically. Use a .pt model (.engine files are Jetson-specific).
# =============================================================================
