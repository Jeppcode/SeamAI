# thesis_experiments

The experiments behind a KTH master's thesis (2026): predicting
whether a pedestrian approaching a door intends to **enter** or just **pass by**,
and checking whether such a model can run in real time on edge hardware.

This folder is the reproducible experiment pipeline. It sits alongside
`data_collection_v2/`, the new data-collection pipeline for future data. Nothing
here changes the original `automatic_door/` scripts.

The datasets and generated results/plots are **not** included (they are large
and contain identifiable video). Everything needed to regenerate them from raw
recordings is here.

```
  utils/              common code: feature extraction, datasets, models

  demo_open_house/     standalone open-house attention demo (with a trained model)
```

Each folder has its own README with the details.

## The research questions

- **RQ1 — Which visual features matter?** Compare a logistic-regression
  baseline, a random forest with SHAP, and an MLP/GRU/CNN ablation over
  trajectory and body-pose features. See `1_RQ1_feature_importance/`.
- **RQ2 — Can it run on the edge?** Benchmark the classifier and the camera
  perception frontends on the target hardware against a 30 FPS budget. See
  `2_RQ2_RQ3_edge_deployment/` (steps 1–8).
- **RQ3 — What is the end-to-end latency?** Measure full system latency from a
  live camera. See `2_RQ2_RQ3_edge_deployment/` (step 9).

## Pipeline order

The folders are numbered in the order they run. Each stage reads what the
previous one produced.

```bash
# 0. Build the cleaned dataset (one time)
#    Put raw recordings in DataOriginal/ first (or pass --src).
python 0_data_cleaning/build_masterdata.py
python 0_data_cleaning/extract_pose.py

# 1. RQ1 feature-importance experiments
python 1_RQ1_feature_importance/baseline_logreg/train_logreg.py
python 1_RQ1_feature_importance/baseline_rf_shap/rf_feature_importance.py
python 1_RQ1_feature_importance/neural_network_ablation/train_ablation.py
python 1_RQ1_feature_importance/neural_network_ablation/select_gru_finalists.py

# 2. RQ2/RQ3 edge benchmarks (run from inside the folder)
cd 2_RQ2_RQ3_edge_deployment
python run_all.py
```

Step 2 reuses the RQ1 ablation results, so run RQ1 before RQ2. Each script says
on screen which inputs it needs and where it writes its outputs.

For a full step-by-step walkthrough on the Jetson — setup, where to put the data,
and every command — see `RUN_ON_JETSON.md`.

## Setup

The code runs on macOS, x86 Linux, and the lab Jetson. On a laptop, a normal
virtual environment is enough:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python check_env.py        # confirms the key packages import
```

On the Jetson, install PyTorch from NVIDIA's wheels first — see the notes at the
top of `requirements.txt`. The TensorRT engine files used by the edge benchmarks
are built on the Jetson itself; they are device-specific and are not committed.

See `USAGE.md` for full setup (laptop and Jetson) and a per-script guide to the
options each script takes.

## Data and results

Raw recordings live in `DataOriginal/` and the cleaned dataset in `MasterData/`.
Both are empty here — see their READMEs for the expected layout and how to obtain
or rebuild them. To collect more data for these experiments, use the original
collection script `automatic_door/automatic_labeling/data_collection.py` (it
produces the `DataOriginal/` format), then run step 0 to rebuild `MasterData/`.
Results and plots are written into per-experiment `results/` and `figures/`
folders when you run the scripts; they are regenerated, not shipped.

The recordings are video of real people. Handle the data accordingly and keep it
out of version control.
